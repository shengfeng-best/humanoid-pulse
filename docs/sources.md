# 来源质量闸门

人形脉冲只收录**可回溯、可核验**的一手或权威二手来源。每条进入 `issue.md` 的条目必须能点开原文 URL，且来源类别符合下列白名单之一。

## 允许收录

### 1. 政府与部委

- 国务院、各部委、地方政府官网发布的政策、通知、试点方案、权威通稿
- 示例域名：`gov.cn`、`miit.gov.cn`、地方政务网

**要求：** `source` 写发布机构全称（如「工信部」）；`url` 指向该机构页面，而非转载摘要。

### 2. 央媒与一线专业科技媒体

- 人民日报、新华社、央视等央媒，以及具备编辑流程、可核查引用的专业科技媒体
- 若报道引用官方文件或厂商新闻稿，**优先链接一手出处**；仅在无法获得一手时使用媒体稿，并在短评中说明事实来源

**要求：** `source` 写媒体名；`url` 为可打开的稿件页。

### 3. 顶会、顶刊与预印本

- NeurIPS、ICRA、CoRL、RSS 等顶会论文或官方会议材料
- Nature、Science 及子刊、IEEE 等同行评议出版物
- arXiv、OpenReview 等预印本平台的**官方条目页**（非第三方摘要）

**要求：** `source` 写会议/期刊/平台名；`url` 为论文或官方材料页。

### 4. 厂商、实验室与开源项目官方渠道

- 一线机器人/具身智能厂商、研究实验室的**官方**新闻稿、博客、产品页
- GitHub Release、官方文档站点、项目 README 中的正式发布

**要求：** `source` 写机构或项目名；`url` 为该机构自有域名下的页面。

## 禁止收录

以下类型**不得**进入周刊，即使标题吸睛：

| 类型 | 说明 |
|------|------|
| 无出处传闻 | 社交媒体流言、群聊转述、无法核验的「据悉」 |
| 营销软文 | 无事实核验的品牌通稿堆砌、通稿农场改写 |
| 聚合站二手改写 | 标题党资讯站、RSS 洗稿站，且**无法回链**至一手原文 |
| 断链或占位 URL | 构建前须确认链接可访问；`--strict` 模式下不可达则失败 |

常见需警惕的聚合/转载站（非穷举）：各类「快讯」「机器人资讯大全」类门户，若仅为改写且无原始出处链接，一律不用。

## 采编操作清单

出刊前，编辑对**每一条**自检：

1. **可点回原文** — `url` 在浏览器中打开为正确页面，非 404 或跳转首页
2. **优先一手** — 同一事件若同时存在部委通稿与媒体解读，链接通稿
3. **来源字段诚实** — `source` 与 URL 域名/机构一致，不张冠李戴
4. **短评基于原文** — 2–4 句短评中的事实可在原文中找到依据；判断须标注为编辑观点
5. **配图有据** — 仅使用来源官方图、可合理引用的现场图或自有素材；存于 `issues/.../assets/`
6. **体量达标** — 国内政策 3–4 条、国外前沿 3–4 条、重点新闻 6–8 条（见 [issue-schema.md](./issue-schema.md)）
7. **拿不准就删** — 无法确认真实性或出处的条目，宁可少发一条也不凑数

## 与构建工具的关系

- `parse_issue.py` 校验字段完整性（标题、来源、URL、短评）
- `build.py --strict` 可选对每个 URL 做 HEAD 检查；默认不阻断，由人工闸门负责质量
- 本文件是**内容**质量规范；技术 schema 见 [issue-schema.md](./issue-schema.md)
